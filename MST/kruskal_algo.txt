Kruskal’s Algorithm

Kruskal’s Algorithm is a greedy algorithm used to find the Minimum Spanning Tree (MST) of a connected, weighted, undirected graph.

A Minimum Spanning Tree is a tree that:

connects all vertices

has no cycles

has minimum total edge weight

How Kruskal’s Algorithm Works

Kruskal’s Algorithm works by selecting edges in increasing order of weight.

At each step:

choose the smallest edge

add it only if it does not form a cycle

repeat until V - 1 edges are selected

Steps of Kruskal’s Algorithm

Sort all edges in increasing order of weight.

Pick the smallest edge.

Check whether adding it forms a cycle.

If no cycle is formed, include it in the MST.

If a cycle is formed, discard it.

Repeat until V - 1 edges are included in the MST.

Example

Suppose the graph has vertices: A, B, C, D

Edges:

A-B = 1

B-C = 2

C-D = 3

A-C = 4

B-D = 5

Process

Pick A-B = 1

Pick B-C = 2

Pick C-D = 3

Now V - 1 = 3 edges are selected.

MST Edges

A-B = 1

B-C = 2

C-D = 3

Total Cost

1 + 2 + 3 = 6

Time Complexity

Sorting edges: O(E log E)

Cycle checking with Disjoint Set: efficient

Overall complexity: O(E log E)

Applications of Kruskal’s Algorithm

Network design

Road and railway connection

Minimum cost wiring

Water supply systems

Connecting computers in a network

Important Points

Kruskal’s Algorithm is used for weighted, undirected graphs

It follows a greedy approach

It selects edges by smallest weight first

It avoids cycles

It is commonly implemented using Disjoint Set Union (DSU)

Conclusion

Kruskal’s Algorithm is a simple and efficient method to find the Minimum Spanning Tree by choosing the smallest edges one by one while avoiding cycles.
